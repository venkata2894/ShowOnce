# Contributing to ShowOnce

First off, thank you for considering contributing to ShowOnce! It's people like you that make ShowOnce such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Explain which behavior you expected to see and why.

### Suggesting Enhancements
- Check if there's already an issue for it.
- Explain why this enhancement would be useful to most ShowOnce users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Install dependencies: `pip install -r requirements.txt`.
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes: `pytest`.
5. Format your code with `black`.
6. Issue that pull request!

## Development Setup

- Python 3.10+
- Anthropic API Key (for testing analysis)
- `playwright install` (for browser-based tests)

## Style Guide
- Follow PEP 8.
- Use meaningful variable and function names.
- Document new functions and classes.

Thank you!
